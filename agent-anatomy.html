매 모델 호출마다 전송

system prompt + 전체 messages + tool 스펙. 루프가 돌수록 messages가 누적되어 매번 통째로 다시 전송됩니다 — 그래서 컨텍스트 관리·캐싱이 중요.

content block

text — 일반 텍스트 toolUse — 툴 호출(id·name·input) toolResult — 결과 되먹임
Hooks — 루프 곳곳에 손을 넣는 지점
에이전트 생명주기의 정해진 순간마다 이벤트가 발생합니다. 여기에 콜백을 걸어 가드레일·로깅·승인·마스킹을 끼웁니다. from strands.hooks import ...
등록 — HookProvider.register_hooks(registry) 안에서 registry.add_callback(BeforeToolCallEvent, fn), 또는 간단히 agent.add_hook(fn, BeforeToolCallEvent). (구 ...InvocationEvent 이름은 레거시 → 현재는 ToolCall/ModelCall)

정리 — Agent는 Model·Tools·messages·Conversation Manager·Event Loop·Hooks·State/Session의 조합입니다. 대부분은 프레임워크가 관리하고, 우리는 툴을 얹고 hook으로 개입할 뿐. 이 구조를 알면 프롬프트가 실제로 뭘 받고, 어디서 끼어들 수 있는지가 보입니다.
