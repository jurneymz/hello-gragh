적용 방식

  • 입력/출력에 독립 설정 — 콘텐츠 필터는 inputStrength/outputStrength(NONE·LOW·MEDIUM·HIGH).
  • 모델 호출과 분리해 임의 텍스트를 검사하는 ApplyGuardrail API (source: INPUT|OUTPUT)도 있음.
  • Contextual grounding·Automated Reasoning은 응답을 봐야 하므로 사실상 출력 측.
코드로 붙이기 & 위협 모델
가드레일 연결, 사람 승인(HITL) 훅, 외부 자격증명(Outbound Auth)까지 — 그리고 프롬프트 인젝션을 다루는 원칙.
① Bedrock 가드레일 연결 (Strands)from strands import Agent
from strands.models import BedrockModel

model = BedrockModel(
    model_id="anthropic.claude-sonnet-4-...",
    guardrail_id="gr-abc123",
    guardrail_version="DRAFT",
    guardrail_trace="enabled",           # 위반 상세 추적
    guardrail_redact_input=True,        # 차단 시 입력을 히스토리에서 가림(기본 True)
)
agent = Agent(model=model)   # 입력·출력 모두 가드레일 통과
② 민감 행동은 사람 승인 — HITL 훅from strands.hooks import BeforeToolCallEvent

def require_approval(event: BeforeToolCallEvent):
    if event.tool_use["name"] == "send_email":
        event.cancel_tool = "이메일 전송 전 승인이 필요합니다"   # 툴 실행 차단

agent.add_hook(require_approval, BeforeToolCallEvent)

프롬프트 인젝션 위협 모델

✓ 해야 할 것
  • 최소 권한 툴 — 꼭 필요한 것만 (Strands Responsible AI 원칙).
  • 입력 검증 — 툴 인자를 철저히 검사(경로 탈출 등).
  • 양방향 가드레일 — 입력 PROMPT_ATTACK + 출력 필터.
  • 민감 행동은 HITL — 전송·결제·삭제는 사람 승인.
  • 감사 로그 — 민감 작업 기록(CloudTrail).
✗ 하지 말 것
  • 툴이 돌려준 외부 콘텐츠를 "지시"로 신뢰하지 말 것 — 인젝션의 통로.
  • 하나의 에이전트에 외부 읽기 + 민감 시스템 조작 권한을 몰아주지 말 것.
  • 실행 역할에 과도한 IAM 권한을 주지 말 것(호출자보다 커지면 안 됨).
  • 비밀키를 코드/프롬프트에 하드코딩하지 말 것.

⚠️ 검증 노트 (2026-07)

  • AgentCore는 요청마다 전용 microVM으로 세션 격리(완료 시 메모리 소거). 단, 세션↔사용자 매핑은 앱 책임.
  • Automated Reasoning 체크는 GA지만 detect 전용(차단 안 함) — 인젝션 방어는 콘텐츠 필터로.
  • OWASP LLM Top 10(예: 프롬프트 인젝션·과도한 자율성)은 업계 일반 지침으로, 위 공식 문서에 직접 인용된 건 아님.

핵심 — 어느 한 통제도 완벽하지 않기에 겹으로 쌓습니다. 인증(Identity) · 가드레일(양방향) · 훅 개입(HITL) · 최소권한 · 격리 · 감사 — 각각 다른 실패를 막습니다. 그리고 가장 중요한 습관 하나: 모델과 툴 출력을 신뢰할 수 있는 지시로 취급하지 않기.
