에이전트, 이렇게 굴러갑니다

단일 에이전트(ReAct)를 넘어, 여러 에이전트를 엮는 방법·런타임의 구성요소·루프 내부·기억까지. 탭을 눌러 각 개념을 실행해 보세요.

strands.multiagent · 4 patterns
여러 에이전트를 엮는 4가지 방법
Strands가 다중 에이전트를 구성하는 기본기. 넷을 동시에 돌려 흐름의 성격을 비교하세요 — 고정 vs 자율, 순차 vs 허브.
Graph고정 DAG · 결정적
    from strands.multiagent import GraphBuilder b.add_node(a,"x"); b.add_edge("x","y") graph = b.build(); graph(task) # GraphResult
    Swarm자율 핸드오프
      from strands.multiagent import Swarm Swarm([a, b, c])(task) # handoff_to_agent 툴 자동 주입 · 순차 핸드오프
      Workflow순차 파이프라인 · 툴
        from strands_tools import workflow agent.tool.workflow(action="create", tasks=[...]) # 클래스 아님 — 툴(또는 Graph)로 구현
        Agents-as-Tools허브-스포크
          @tool def sub(q): return str(Agent(...)(q)) Agent(tools=[sub]) # 에이전트를 툴처럼

          고르는 기준 — 순서·검증이 중요하면 Graph/Workflow, 다음에 누가 일할지 모델이 판단하게 하려면 Swarm, 전문 에이전트를 기능처럼 불러 쓰려면 Agents-as-Tools.
          Swarm 심화 — 각 에이전트에 handoff_to_agent 툴이 자동으로 주입되고, 에이전트가 이를 호출해 공유된 컨텍스트와 함께 제어권을 넘깁니다. 현재는 순차 핸드오프(한 번에 하나)이며, max_handoffs·max_iterations로 상한을, 반복 핸드오프 감지(repetitive_handoff_detection_window)로 무한 핑퐁을 막습니다.
          위 코드 한 줄들은 2026-07 기준 실제 SDK 시그니처로 검증됨.

          bedrock-agentcore · building blocks
          AgentCore 빌딩블록 맵
          요청 하나가 들어왔을 때 Identity·Runtime·Memory·Gateway·Observability가 어떻게 맞물리는지 경로를 따라가 봅니다.

          요청 경로

            핵심 — 이 블록들은 조합형입니다. 다 쓸 필요 없이, 필요한 것만(예: Runtime + Memory) 골라 붙입니다. 에이전트 코드는 그대로 두고 인프라 걱정을 런타임에 위임하는 게 목적.

            agentic event loop
            Agent Loop 내부 한 사이클
            ReAct의 속을 확대한 그림. 모델이 tool_use를 내면 툴을 실행해 tool_result를 messages에 되먹임하고, 더 부를 툴이 없을 때 최종 텍스트를 냅니다.

            event loop

              핵심 — 반복을 멈추는 건 코드가 아니라 모델의 판단입니다. tool_use 블록이 더 안 나오면 그 자체가 종료 신호.

              short-term vs long-term memory
              Memory 동작
              한 대화 안의 단기 기억(context window, 휘발)과, 세션이 바뀌어도 남는 장기 기억(AgentCore Memory). 이벤트를 저장하고 전략으로 요약해 다음에 되불러옵니다.

              저장 → 요약 → 검색

                핵심 — 단기 기억은 창을 넘으면 사라집니다(점선 블록). 오래 기억해야 할 사실·선호는 장기 저장소에 이벤트로 남기고, 다음 세션에서 관련된 것만 검색해 주입.

                하나로 꿰면 — 단일 에이전트는 루프(03)로 돌고, 여러 개를 엮을 땐 4패턴(01) 중 하나를 고릅니다. 기억은 Memory(04)가, 실행·인증·툴연결·관측은 AgentCore(02)가 맡습니다. 앞서 만든 ReAct vs Graph는 이 지도의 01번 첫 갈림길이었어요.
