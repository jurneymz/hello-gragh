ReAct vs Graph

같은 작업을 줬을 때, 에이전트가 스스로 판단하며 루프를 도는가(ReAct), 아니면 개발자가 그린 고정된 그래프를 따라가는가(Graph). 아래에서 실행해 흐름을 직접 비교하세요.

tip — 여러 번 실행: ReAct는 매번 반복 수가 바뀌고, Graph는 항상 같은 경로

      ReAct — 단일 Agent# 툴만 쥐어주면, 루프는 모델이 알아서
from strands import Agent
from strands_tools import calculator, http_request

agent = Agent(tools=[calculator, http_request])
agent("프랑스와 일본 GDP를 비교해줘")
# reason → tool → observe → … → answer
# 몇 번 돌지는 실행 시점에 결정됨
      Graph — 다중 Agent DAGfrom strands.multiagent import GraphBuilder

b = GraphBuilder()
b.add_node(classifier, "classify")
b.add_node(researcher, "research")
b.add_node(writer,     "write")
b.add_edge("classify", "research")
b.add_edge("research", "write")
graph = b.build()
graph("프랑스와 일본 GDP를 비교해줘")  # 경로 고정

      배포는 둘 다 똑같습니다. ReAct든 Graph든 @app.entrypoint 로 감싸 Bedrock AgentCore Runtime 위에 올리는 방식은 동일해요. 차이는 런타임 안에서 어떻게 실행되느냐일 뿐 — 짧고 개방적인 작업엔 ReAct, 순서·검증이 중요한 다단계 파이프라인엔 Graph가 맞습니다.
